  1. PREREQUISITES
====================================================================

Before you start, make sure you have:
  - Access to your GTM container (GTM-PJV92STC)
  - Access to your GA4 property
  - A GA4 Configuration Tag already set up in GTM
    (if not, see step below)

IF YOU DON'T HAVE A GA4 CONFIGURATION TAG YET:
  1. In GTM, click "Tags" > "New"
  2. Name it: "GA4 - Configuration"
  3. Tag type: "Google Analytics: GA4 Configuration"
  4. Measurement ID: paste your GA4 Measurement ID
     (starts with "G-")
     - Find this in GA4 > Admin > Data Streams >
       select your stream > Measurement ID
  5. Trigger: "All Pages"
  6. Save

Throughout this guide, when a tag says "use your GA4
Measurement ID", you'll enter that same G-XXXXXXXX
value.

the events your website sends. You need one for each

  3. CREATE CUSTOM EVENT TRIGGERS (7 total)
====================================================================

Custom Event Triggers tell GTM "when this specific
event happens on the site, fire a tag." Each one
listens for a specific event name that the website
code sends via dataLayer.push().

For EACH trigger below:
  1. Click "Triggers" in the left sidebar
  2. Click "New"
  3. Name it exactly as shown
  4. Click the trigger configuration box
  5. Choose "Custom Event"
  6. Enter the Event Name exactly as shown
     (case sensitive!)
  7. Leave "This trigger fires on" as
     "All Custom Events"
  8. Save

====================================================================
  4. CREATE BUILT-IN TRIGGERS (2 total)
====================================================================

These use GTM's built-in trigger types -- no code
changes needed on the site.

--- TRIGGER 8: Scroll Depth ---

  1. Click "Triggers" > "New"
  2. Name: "Scroll Depth - 25 50 75 90"
  3. Click the trigger configuration box
  4. Choose "Scroll Depth"
  5. Configure:
     - Vertical Scroll Depths: CHECK this box
     - Percentages: 25, 50, 75, 90
     - (Leave Horizontal unchecked)
     - This trigger fires on: "All Pages"
  6. Save

IMPORTANT: You also need to enable the built-in
"Scroll Depth Threshold" variable:
  1. Go to "Variables"
  2. Click "Configure" at the top of
     "Built-In Variables"
  3. Scroll down to the "Scrolling" section
  4. Check "Scroll Depth Threshold"
  5. Also check "Scroll Depth Units" and
     "Scroll Direction" while you're here


--- TRIGGER 9: Outbound Link Clicks ---

  1. Click "Triggers" > "New"
  2. Name: "Click - Outbound Links"
  3. Click the trigger configuration box
  4. Choose "Click - Just Links"
  5. Check "Wait for Tags" (max wait: 2000ms)
  6. Check "Check Validation"
  7. This trigger fires on: "Some Link Clicks"
  8. Condition:
     "Click URL" > "does not contain" >
     "futurlabs.dev"
     (Replace with your actual domain if different)
  9. Save

IMPORTANT: Enable the built-in "Click URL" variable:
  1. Go to "Variables" > "Configure"
     (Built-In Variables)
  2. Under "Clicks", check "Click URL" and
     "Click Text"

====================================================================

Each tag sends an event to GA4 when its trigger
fires. The event parameters carry the data that
makes your reports useful.

For EACH tag below:
  1. Click "Tags" > "New"
  2. Name it exactly as shown
  3. Click the tag configuration box
  4. Choose "Google Analytics: GA4 Event"
  5. Measurement ID: enter your GA4 Measurement ID
     (G-XXXXXXXXXX) OR select your GA4
     Configuration Tag
  6. Event Name: enter exactly as shown
  7. Expand "Event Parameters" and click
     "Add Row" for each parameter
  8. Set the trigger as specified
  9. Save

====================================================================
  6. REGISTER CUSTOM DIMENSIONS IN GA4 (11 total)
====================================================================

Custom dimensions let your event parameters show
up in GA4 reports and explorations. Without this
step, the data is collected but you can't see it
in reports.

For EACH dimension:
  1. Go to GA4 (analytics.google.com)
  2. Click Admin (gear icon, bottom left)
  3. Under "Data display", click
     "Custom definitions"
  4. Click "Create custom dimension"
  5. Fill in as shown below
  6. Click "Save"

  +----+------------------+------------------+-------+
  | #  | Dimension Name   | Event Parameter  | Scope |
  +----+------------------+------------------+-------+
  | 1  | CTA Location     | cta_location     | Event |
  | 2  | CTA Text         | cta_text         | Event |
  | 3  | Pricing Tier     | pricing_tier     | Event |
  | 4  | Pricing Range    | pricing_range    | Event |
  | 5  | Form Source      | form_source      | Event |
  | 6  | Form Step Number | step_number      | Event |
  | 7  | Form Step Name   | step_name        | Event |
  | 8  | Project Type     | project_type     | Event |
  | 9  | Budget Range     | budget           | Event |
  | 10 | Company Size     | company_size     | Event |
  | 11 | Referral Source   | referral         | Event |
  +----+------------------+------------------+-------+

  "Scope" should always be "Event" for all of these.

  NOTE: GA4 free properties allow up to 50 custom
  dimensions. You're using 11.


====================================================================
  7. MARK CONVERSIONS IN GA4
====================================================================

Mark "generate_lead" as a key event (conversion)
so GA4 highlights it in reports and you can use it
for Google Ads optimization.

  1. In GA4, go to Admin > Events
  2. Wait for "generate_lead" to appear in the
     events list (it will show up after the first
     form submission with tracking live)
  3. Toggle the "Mark as key event" switch next
     to "generate_lead"

  If you don't see it yet, you can create it
  manually:
  1. In Admin > Events, click "Create event"
  2. Custom event name: generate_lead
  3. Save, then toggle "Mark as key event"


====================================================================
  8. TEST EVERYTHING WITH PREVIEW MODE
====================================================================

BEFORE publishing, always test! GTM Preview Mode
shows you exactly what fires.

STEP 1: OPEN PREVIEW MODE
  1. In GTM, click "Preview" (top right corner)
  2. Enter your website URL
     (e.g., https://futurlabs.dev or
     http://localhost:3000)
  3. Click "Connect"
  4. Your site opens in a new tab with
     Tag Assistant connected

STEP 2: TEST EACH EVENT

  Test CTA Click:
  - Click "Get a Free SaaS Audit" in the hero
  - In Tag Assistant, you should see "cta_click"
    event fire
  - Click on it and verify:
    cta_location = "hero"
    cta_text = "Get a Free SaaS Audit"
  - Close the modal
  - Now click "Book a Discovery Call" in the nav
  - Verify cta_location = "nav"
  - Scroll down and test the footer CTA too

  Test Form Open:
  - Click any CTA that opens the contact form
  - Verify "form_open" event fires with the
    correct form_source
  - If you clicked from pricing, verify
    pricing_tier is populated

  Test Form Steps:
  - Fill out step 1 (name, email, company) and
    click Next
  - Verify "form_step" fires with
    step_number = 1, step_name = "about_project"
  - Continue through all steps

  Test Form Submit:
  - Complete the full form and submit
  - Verify "generate_lead" fires with all
    parameters populated
  - Check: project_type, budget, company_size,
    timeline, referral

  Test Form Abandon:
  - Open the form, fill out step 1, advance to
    step 2
  - Close the modal (X button or click backdrop)
  - Verify "form_abandon" fires with
    last_step = 1

  Test Pricing Click:
  - Click "Get Started" on any pricing card
  - Verify both "pricing_click" AND "cta_click"
    fire
  - Check pricing_tier and pricing_range values

  Test FAQ Toggle:
  - Click on any FAQ question to open it
  - Verify "faq_toggle" fires with question text
  - Click the same question to close it -- verify
    it does NOT fire again

  Test Scroll Depth:
  - Scroll slowly down the page
  - Verify "scroll_depth" fires at
    25%, 50%, 75%, 90%

  Test Outbound Click:
  - Click a social media link (LinkedIn or
    Twitter) in the footer
  - Verify "outbound_click" fires with link_url

STEP 3: CHECK FOR ERRORS
  - In Tag Assistant, look for any red error
    indicators
  - Click on any event to see its details and
    verify parameter values
  - If a tag didn't fire, check that the trigger
    event name matches exactly


====================================================================
  9. PUBLISH YOUR CONTAINER
====================================================================

Once everything tests correctly:

  1. Click "Submit" (top right in GTM)
  2. Version Name:
     "Analytics tracking - CTA, form funnel,
     FAQ, scroll depth"
  3. Version Description (optional):
     "Added tracking for: CTA clicks with
     location, 4-step form funnel
     (open/step/submit/abandon), pricing tier
     clicks, FAQ opens, scroll depth, and
     outbound link clicks. 9 GA4 event tags
     total."
  4. Click "Publish"

Your tracking is now live!


====================================================================
  QUICK REFERENCE: WHAT EACH EVENT TELLS YOU
====================================================================

  cta_click
    "Which buttons do people click, and where
    on the page?"
    Look at: cta_location breakdown --
    hero vs nav vs footer vs pricing

  form_open
    "How many people open the contact form, and
    where did they come from?"
    Look at: form_source -- which CTAs actually
    drive form opens

  form_step
    "How far do people get in the form?"
    Build a funnel: step 0 > 1 > 2 > 3 --
    spot the biggest drop-off

  generate_lead (CONVERSION)
    "How many leads did we get, and what do
    they want?"
    Look at: budget, project_type, company_size
    -- understand your pipeline

  form_abandon
    "Where do people give up on the form?"
    Look at: last_step_name -- if most abandon
    at budget_timeline, maybe pricing is
    scaring people off

  pricing_click
    "Which pricing tiers get the most interest?"
    Compare: Discovery vs MVP vs Full Platform
    vs Growth Retainer

  faq_toggle
    "What questions do visitors need answered?"
    If one FAQ gets 10x the clicks, promote
    that answer higher on the page

  scroll_depth
    "How far down the page do people scroll?"
    If 70% don't reach pricing, consider
    moving it up

  outbound_click
    "Are people clicking our social links?"
    Track which external links get engagement


====================================================================
  REPORTING TIPS
====================================================================

FUNNEL EXPLORATION (see where people drop off):
  1. GA4 > Explore > Create new exploration
  2. Choose "Funnel exploration"
  3. Add steps:
     Step 1: form_open
     Step 2: form_step (filter: step_number = 1)
     Step 3: form_step (filter: step_number = 2)
     Step 4: form_step (filter: step_number = 3)
     Step 5: generate_lead
  4. This shows your exact conversion funnel
     drop-off

CTA COMPARISON:
  1. GA4 > Reports > Engagement > Events
  2. Click on "cta_click"
  3. Add secondary dimension: "CTA Location"
  4. Now you can see which page locations drive
     the most clicks

PRICING INTEREST:
  1. GA4 > Explore > Free form exploration
  2. Rows: Pricing Tier
  3. Values: Event count
  4. Filter: Event name = pricing_click
  5. See which tiers attract the most interest
